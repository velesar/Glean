## API Endpoints

### Projects
```
GET    /api/projects              # List all projects
POST   /api/projects              # Create project
GET    /api/projects/:id          # Get project details
PUT    /api/projects/:id          # Update project config
DELETE /api/projects/:id          # Delete project
```

### Tools
```
GET    /api/tools                 # List tools (with filters)
GET    /api/tools/:id             # Get tool details
PUT    /api/tools/:id/status      # Update status (approve/reject)
GET    /api/tools/:id/claims      # Get claims for tool
```

### Jobs
```
GET    /api/jobs                  # List jobs
POST   /api/jobs/scout            # Start scout job
POST   /api/jobs/analyze          # Start analyzer job
POST   /api/jobs/curate           # Start curation job
POST   /api/jobs/update           # Start update check
GET    /api/jobs/:id              # Get job status
DELETE /api/jobs/:id              # Cancel job
```

### Reports
```
GET    /api/reports/weekly        # Get weekly digest
GET    /api/reports/changelog     # Get changelog
GET    /api/reports/index         # Get tools index
GET    /api/stats                 # Get pipeline stats
```

### Activity
```
GET    /api/activity              # Get activity log
```

## Tech Stack

### Backend
- **FastAPI** - Modern Python web framework
- **SQLite** - Existing database
- **Background Tasks** - For long-running jobs
- **Pydantic** - Request/response validation

### Frontend
- **React 18** - UI framework
- **TypeScript** - Type safety
- **Tailwind CSS** - Styling
- **React Query** - Data fetching
- **React Router** - Navigation

## Implementation Plan

1. **Phase 1: API Backend**
   - Set up FastAPI app structure
   - Implement core endpoints (tools, stats)
   - Add job management with background tasks

2. **Phase 2: Basic Frontend**
   - Dashboard with pipeline stats
   - Tools list with filtering
   - Basic review interface

3. **Phase 3: Projects & Config**
   - Multi-project support
   - Scout configuration UI
   - Project-specific settings

4. **Phase 4: Jobs & Activity**
   - Real-time job progress
   - Activity log
   - Job cancellation

5. **Phase 5: Polish**
   - Responsive design
   - Error handling
   - Loading states
